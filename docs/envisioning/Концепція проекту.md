# Концепція проекту

Етап: *Аналіз проєкту (Envisioning)*

Проект: *"Weather Today"*

## 1. Необхідність проекту
### 1.1 Обгрунтування необхідності
Проект спрямований на полегшеня занаходження прогнозу погоди на день, за допомогою чат-бота у застосунку телеграм.

Чат-бот повиннен кожного ранку надсилати повідомлення з прогнозом на день, користувач може у будь який момент дізнатися погоду зараз, на наступний день, та на три дні у перед, також передбачена англіська локалізація. Таким чином чат-бот призваний полегшувати рутину.

### 1.2 Бачення проекту

протягом місяця розробити чат-бот у застосунку телеграм, яки буде надсилати погоду кожного ранку та показувати прогноз погоди зараз, на наступний день, та на три дні у перед, та англіська локалізація.

## Концепція рішення
### 2.1 Цілі та завдання

**Мета 1: реалізувати базову функціональність метео-сервісу**
- погода на день
- погода на наступний день
- погода на три дні
- погода зараз
- реалізувати англіську локалізацію

**Мета 2: реалізувати відправку повідомлень кожного ранку**
- забезпечити моніторинг часу *cron*
- зробити ідентифікацію *часового поясу*
- впровадити своєчасне відправлення повідомлень у кожному часовому поясі

**Мета 3: забезпечити зручний інтерфейс**
- зробити гарно відформатований текст повідомлень
- забезпечити гарну інтерактивність продукту
- додати два види кнопок

### 2.2 Аналіз використання
**Групи користувачів**
- кур'єри доставки
- організатори заходів на відкритому повітрі
- звичайні люди
- логісти
- туристи

**Основні сценарії використання**
- визначення погоди для продуктивної роботи
- визначення погодних умов для проведення заходів на відкритому повітрі
- визначення погоди для піших прогулянок
- визначення погодних умов для туризму

## 3. Стратегії проектування
### 3.1 Стратегія архітектурного проектування
Сервіс реалізується як чат-бот у застосунку телеграм з використанням відкритого АРІ для отримання актуальних даних про погоду з методом проектування архитектури **Layered Architecture** (Багатошарова архітектура) **з чітким розділенням обов'язків**:
- Presentation Layer — Telegram Interface (через grammY).
- Business Logic Layer — Weather Today Bot (основний код на Node.js).
- Data Access Layer — робота з MongoDB та Redis.

### 3.2 Стратегія технологічного проектування
**Основні технології**
- **Node.js** - середовище для написання бізнеслогіки
- **GrammY.js** - бібліотека для взаємодії з АРІ чат-ботів телеграм
- **Axios** - для взаємодії з відкритим АРІ для отримання даних про погоду
- **Mongoose** - для взаємодії з Бд MongoDB
- **node-redis** - для взаємодії з кешуванням Redis
- **winston-logerr** - для логування та відстеження помилок при роботі сервісу
- **node-cron** - для відстеження поточного часу

**Бази даних**
- **MongoDB** - для зберігання даних користувачів
- **Redis** - для кешування даних

**Відкритий АРІ**
У проекті буде використаний відкритий АРІ від weatherapi.com для відстеження актуальних метео даних
