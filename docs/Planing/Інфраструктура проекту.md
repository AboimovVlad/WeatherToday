# Інфраструктура та розгортання
Етап: *планування проєкту (Planing)*

Проект: *"Weather Today"*

## 1. Технологічний стек та середовище
- У проекті використовується середовище виконання коду **node.js**, для розробки легковагомого чат-боту.
- За платформу чат-бота взято **телеграм**.
- Хостингом для сервісу буде використано **server.ua/vps (1 ядро, 1ГБ ОЗУ, 10ГБ жорсткий диск)**
- За ОС хостингу буде використано **Linux Ubuntu** за для гарної підримки **Docker.**

*\*Таким чином забезпечується стабільна робота чат-бота на віддаленому сервері*

## 2. Керування контейнерами (Docker)
Для забезпечення ізоляції сервісу він буде запущений за допомогою **Docker**, разом з сервісом буде запущено **noSQL Бд Redis** для кешування даних які часто використовуються користувачами, запускати та оркеструвати контейнери буде **Doker-compouse**.
- Doker-compouse: виконує роль оркестрації контейнерів з сервісом та noSQL Бд Redis
- сервіс **app**: контейнер з кодом сервісу на платформі node.js
- сервіс **redis**: контейнер з офіційним образом Redis для кешування даних АРІ

*\*Завдяки Doker можно зручно запускати сервіс будь де та будь коли*

## 3. Бази даних та зовнішні сервіси
У проекті буде використано noSQL бази даних, для швидкого та зручного зберігання даних користувачів, логування, сесій користувачів, розсилок погоди кожного дня та для зберігання часто запитуваних даних про прогноз на зараз.

- **Primary DB (MongoDB):** буде використана для зберігання даних о користувачах, зберігання сесій користувачів, логування та постійне зберігання кому треба зробити ранкову розсилку повідомлень з прогнозом.
- **Cache DB (Redis):** буде використано для кешування постійних запитів погоди на данний момент це забезпечит зменшення навантаження на АРІ який забезпечує сервіс даними, також кешуватися будуть дані користувачів яким потрібно надіслати ранкове повідомлення з прогнозом на день.
- **External API:** у проекті буде використано weatherapi.com для забезпечення актуальними метео даними чат-бота, взаємодія між сервісами відбувається за протоколом HTTPS, це дозволяє безпечно отримуати дані та не хвилюватися за їх цілісність.

*таким чином забезпечується велика стресо стійкість від користувачів та мінімізує перенавантаження сервера*

## 4. Конфігурація (Environment Variables)
Список заміних середовища для проекту
- **BOT_TOKEN** — токен від BotFather.
- **MONGO_URI** — рядок підключення до Atlas.
- **REDIS_URL** — адреса локального Redis.
- **WEATHER_API_KEY** — ключ від weatherapi.com.

## 5. Логування даних та їх збереження
Для логування у проєкті буде використано Winston для зручного моніторингу та відстеження помилок підчас використання сервісу.

- Логування Winston: забеспечить зручне відстеження помилок підчас використання сервісу та моніторингу роботи.
- Якщо сервіс по той чи інщій причині ліг: виокристаємо Restart Policy у Docker (restart: unless-stopped)

*завдяки такому рішеню буде легко робити моніторинг севісу*

## 6. Стратегія розгортання (Deployment Flow)
1. Push коду в репозиторій.
2. Pull на сервері.
3. Запуск `docker-compose up --build -d.`
