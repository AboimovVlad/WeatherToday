# Архітектура програмного забезпечення

Етап: *планування проєкту (Planing)*

Проект: *"Weather Today"*

## 1. Опис архітектурного підходу
### Архітектурний підхід
У проекті планується використати **Layered Architecture**, бо завдяки йому можна зручно розділити логіку застосунку на різні шари, та використати підхід роботи севісу Event-Driven, він дозволяє не перенавантажувати сервер на якому працю сервіс:
- **Layered Architecture (Багатошарова архітектура)**: буде використано такі шари, як **Presentation (Bot)** тут буде знадитися обробка команд та перенаправлення на -> **Business Logic (Services)** тут відбувається валідація повідомлень, отримання даних користувача, виклики бази даних, щоб база даних працювала потрібен -> **Data Access (Repository)** цей шар дозволяє зручно звертатися до Бд та отримувати з неї дані або записувати їх у Бд.
- **Event-Driven (Подійно-орієнтований підхід):** завдяки такому підходу чат-бот не працює постійно, а тільки реагує на події (отримання повідомлень від користувачів, Cron завдяки якому бот надсилає кожного ранку користувачам повідомлення за прогнозом на день).

*\*таким чином забезпечується максимально зручна та масштабована архітектура проекту.*
